### Question 1: Installing Extensions
What is the keyboard shortcut to open the Extensions view?

A) `Cmd+Shift+E` (Mac) / `Ctrl+Shift+E` (Windows/Linux)  
B) `Cmd+Shift+X` (Mac) / `Ctrl+Shift+X` (Windows/Linux)  
C) `Cmd+Shift+P` (Mac) / `Ctrl+Shift+P` (Windows/Linux)  
D) `Cmd+E` (Mac) / `Ctrl+E` (Windows/Linux)

### Question 2: Settings
Where are user settings stored in VS Code?

A) In the project folder only  
B) In a `settings.json` file in your user directory  
C) In the VS Code installation directory  
D) Settings cannot be saved  

---

### Question 3: Keyboard Shortcuts
How do you open the Keyboard Shortcuts editor?

A) `Cmd+K Cmd+S` (Mac) / `Ctrl+K Ctrl+S` (Windows/Linux)  
B) `Cmd+Shift+K` (Mac) / `Ctrl+Shift+K` (Windows/Linux)  
C) `F1` then type "shortcuts"  
D) Both A and C  

---

### Question 4: Settings Scope
What is the difference between User settings and Workspace settings?

A) There is no difference  
B) User settings apply globally; Workspace settings apply only to the current project  
C) Workspace settings apply globally; User settings apply only to the current project  
D) User settings are read-only  

---

## Answers

<details>
<summary>Click to reveal answers</summary>

### Answer 1: B
`Cmd+Shift+X` (Mac) or `Ctrl+Shift+X` (Windows/Linux) opens the Extensions view where you can search, install, and manage extensions.

### Answer 2: B
User settings are stored in a `settings.json` file in your user directory. On Mac: `~/Library/Application Support/Code/User/settings.json`

### Answer 3: D
Both methods work: `Cmd+K Cmd+S` / `Ctrl+K Ctrl+S` is the direct shortcut, and you can also use the Command Palette (F1) and type "shortcuts".

### Answer 4: B
User settings apply globally to all VS Code instances, while Workspace settings apply only to the current project and override user settings.
