## Questions

### Question 1: Git Integration
What does the "M" indicator next to a file in the Source Control view mean?

A) The file is missing  
B) The file has been modified  
C) The file has been moved  
D) The file has merge conflicts  

---

### Question 2: Debugging
What key do you press to set or remove a breakpoint on the current line?

A) `F5`  
B) `F9`  
C) `F10`  
D) `F12`  

---

### Question 3: IntelliSense
What triggers IntelliSense (code completion) in VS Code?

A) Only `Ctrl+Space`  
B) Typing a dot (.) after an object  
C) Both typing and `Ctrl+Space`  
D) IntelliSense is always visible  

---

### Question 4: Code Snippets
What is the file extension for custom user snippets in VS Code?

A) `.snippet`  
B) `.json`  
C) `.code-snippets`  
D) `.vscode`  

---

### Question 5: Tasks
Where are task configurations stored in a VS Code project?

A) `package.json`  
B) `.vscode/tasks.json`  
C) `settings.json`  
D) `workspace.json`  

---

## Answers

<details>
<summary>Click to reveal answers</summary>

### Answer 1: B
"M" stands for Modified - the file has been changed but not yet staged or committed in Git.

### Answer 2: B
`F9` sets or removes a breakpoint on the current line. `F5` starts debugging, `F10` steps over, and `F12` goes to definition.

### Answer 3: C
IntelliSense can be triggered both automatically (like typing a dot after an object) and manually with `Ctrl+Space`.

### Answer 4: C
Custom user snippets are stored in `.code-snippets` files, which use JSON format to define snippet templates.

### Answer 5: B
Task configurations are stored in `.vscode/tasks.json` within your project directory.

</details>
