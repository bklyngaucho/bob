### Question 4: Editor Groups
What happens when you drag a file tab to the side of the editor?

A) The file is deleted  
B) The file is closed  
C) A new editor group is created with a split view  
D) The file is moved to a different folder  

---

### Question 5: Terminal Management
Which of the following is TRUE about VS Code's integrated terminal?

A) You can only have one terminal open at a time  
B) You can split terminals and have multiple instances  
C) The terminal cannot run shell commands  
D) The terminal is read-only  

---

## Answers

<details>
<summary>Click to reveal answers</summary>

### Answer 1: C
VS Code workspace files use the `.code-workspace` extension. These files store workspace configuration including folders and settings.

### Answer 2: A
`Cmd+\` (Mac) or `Ctrl+\` (Windows/Linux) splits the editor vertically, creating a side-by-side view.

### Answer 3: B
`` Ctrl+` `` (backtick) opens a new integrated terminal. This shortcut works on both Mac and Windows/Linux.

### Answer 4: C
Dragging a file tab to the side of the editor creates a new editor group, allowing you to view multiple files side by side.

### Answer 5: B
You can split terminals and have multiple terminal instances running simultaneously, each in its own tab or split pane.
