## Questions

### Question 1: Interface Components
What is the primary purpose of the Activity Bar in VS Code?

A) To display file contents  
B) To switch between different views like Explorer, Search, and Extensions  
C) To show keyboard shortcuts  
D) To manage terminal sessions  

---

### Question 2: Quick Open
What is the keyboard shortcut to open the Quick Open dialog?

A) `Cmd+Shift+P` (Mac) / `Ctrl+Shift+P` (Windows/Linux)  
B) `Cmd+P` (Mac) / `Ctrl+P` (Windows/Linux)  
C) `Cmd+O` (Mac) / `Ctrl+O` (Windows/Linux)  
D) `Cmd+K` (Mac) / `Ctrl+K` (Windows/Linux)  

---

### Question 3: File Management
Which of the following is NOT a way to create a new file in VS Code?

A) Right-click in Explorer and select "New File"  
B) Use `Cmd+N` / `Ctrl+N`  
C) Use the File menu  
D) Double-click in the Explorer panel  

---

### Question 4: Workspace
What is a workspace in VS Code?

A) A single file open in the editor  
B) A collection of one or more folders/projects  
C) The terminal window  
D) The settings panel  

---

### Question 5: Side Bar
Which view in the Side Bar allows you to search across all files in your project?

A) Explorer  
B) Search  
C) Source Control  
D) Extensions  

---

## Answers

<details>
<summary>Click to reveal answers</summary>

### Answer 1: B
The Activity Bar switches between different views like Explorer, Search, Source Control, Run and Debug, and Extensions.

### Answer 2: B
`Cmd+P` (Mac) or `Ctrl+P` (Windows/Linux) opens Quick Open for fast file navigation.

### Answer 3: D
Double-clicking in the Explorer panel doesn't create a new file. The other three options are valid ways to create files.

### Answer 4: B
A workspace is a collection of one or more folders/projects that you're working on together.

### Answer 5: B
The Search view (magnifying glass icon) allows you to search across all files in your project.
