## Practice with the Sample Project

The [sample-project](../sample-project/) folder contains a complete project with:
- JavaScript files with practice comments
- Intentional bugs for debugging practice
- HTML/CSS for Emmet practice
- Test files for TDD practice
- VS Code configuration examples

### Sample Project Exercises

1. **File Navigation Practice**
   - Use `Cmd+P` to quickly open files
   - Practice Go to Definition on functions
   - Use breadcrumbs to navigate

2. **Multi-Cursor Practice**
   - Open `src/utils.js`
   - Practice adding multiple cursors
   - Refactor variable names

3. **Debugging Practice**
   - Open `src/calculator.js`
   - Find and fix the intentional bugs
   - Use breakpoints and debug console

4. **Git Practice**
   - Make changes to files
   - Stage and commit changes
   - View git history

5. **Search Practice**
   - Search for "TODO" comments
   - Use regex to find patterns
   - Replace across multiple files

## How to Use These Exercises

1. **Follow the modules in order**: Start with Module 1 and progress through
2. **Complete exercises as you go**: Each module has practice exercises
3. **Use the sample project**: Apply skills to real code
4. **Experiment freely**: Try variations and explore features
5. **Practice regularly**: Repetition builds muscle memory

## Tips for Success

- **Take your time**: Don't rush through exercises
- **Experiment**: Try different approaches
- **Use keyboard shortcuts**: Build muscle memory
- **Ask Bob for help**: Use Ask mode to understand concepts
- **Practice daily**: Even 10 minutes helps

## Quick Practice Ideas

### 5-Minute Drills

1. **Speed Navigation**
   - Open 5 files using `Cmd+P`
   - Jump between them with `Ctrl+Tab`
   - Close all with `Cmd+K W`

2. **Multi-Cursor Speed**
   - Select a word
   - Add next occurrence (`Cmd+D`)
   - Edit all at once

3. **Search Mastery**
   - Search for a pattern
   - Replace in multiple files
   - Use regex for complex patterns

### 15-Minute Projects

1. **Refactoring Exercise**
   - Open sample project
   - Refactor a function
   - Run tests to verify

2. **Git Workflow**
   - Create a feature branch
   - Make changes
   - Commit and push

3. **Debugging Session**
   - Set breakpoints
   - Step through code
   - Fix a bug
